## 概述

### 一句话理解

异步编程就是**「等外卖的时候去做别的事」**——当程序在等待网络请求、文件读写时，不傻等，而是去干其他活，等外卖到了再回来接着吃。

### 为什么需要异步编程？

先看一个生活中的例子：

```
同步方式（傻等）:
  点外卖 → 等待30分钟（什么都不做）→ 吃饭
  总耗时：30分钟

异步方式（聪明等）:
  点外卖 → 趁等待的时间写代码、看电影、做家务 → 外卖到了 → 吃饭
  总耗时：30分钟（但你还做了很多事！）
```

程序也一样。网络请求、数据库查询、文件读写这些操作，CPU 大部分时间都在**干等**。异步编程就是利用这些等待时间去做别的事。

### 三种并发模型对比

| 模型 | 原理 | 适合什么 | 切换成本 | 类比 |
|------|------|---------|---------|------|
| 多线程 | 操作系统调度，共享内存 | CPU 密集型 | 高 | 多个厨师共用一个厨房 |
| 多进程 | 独立内存空间 | CPU 密集型 | 很高 | 多个厨房各自独立 |
| **异步** | 单线程协作式，主动让出 | **IO 密集型** | 极低 | 一个厨师做多道菜，等水开时切菜 |

> **异步最适合 IO 密集型任务**（网络请求、文件读写、数据库查询），不适合 CPU 密集型任务（大量计算）。

### Python 异步演变史

```
Python 2:    回调地狱（callback hell，代码一层套一层）
    ↓
Python 3.3:  yield from（用生成器模拟协程）
    ↓
Python 3.4:  asyncio 库 + @asyncio.coroutine
    ↓
Python 3.5:  async / await 语法（原生协程，终于好用了！）
    ↓
Python 3.7+: asyncio 高级 API 完善
    ↓
Python 3.11+: TaskGroup、性能大幅优化
```

## 事件循环（Event Loop）

### 什么是事件循环？

事件循环是异步编程的**核心引擎**，就像一个**交通指挥员**——它盯着所有任务，谁 ready 了就让谁跑，谁在等待就先跳过。

### 工作原理

```
┌─────────────────────────────────────────┐
│              Event Loop                  │
│         （交通指挥员）                    │
│                                          │
│   ┌──────┐    ┌──────┐    ┌──────┐      │
│   │ Task1│    │ Task2│    │ Task3│       │
│   │ 运行 │    │ 等待 │    │ 就绪 │       │
│   └──┬───┘    └──┬───┘    └──┬───┘      │
│      │  await    │           │           │
│      ▼           ▼           ▼           │
│   ┌──────────────────────────────┐      │
│   │       IO 等待队列             │      │
│   │   (epoll/kqueue/IOCP)        │      │
│   └──────────────────────────────┘      │
└─────────────────────────────────────────┘
```

**它做的事很简单：**
1. 从就绪队列里取一个任务执行
2. 任务遇到 `await`（需要等待）→ 挂起它，去执行下一个就绪任务
3. 等待的事情完成了 → 把任务放回就绪队列
4. 循环往复

### 运行事件循环

```python
import asyncio

async def main():
    print("Hello")
    await asyncio.sleep(1)    # 模拟等待1秒
    print("World")

# Python 3.7+ 推荐方式
asyncio.run(main())
```

> `asyncio.run()` 做了三件事：创建事件循环 → 运行协程 → 运行完清理。每个线程同时只能有一个事件循环。

### 事件循环的生命周期

```python
# asyncio.run() 的简化等价实现
def my_run(main_coroutine):
    loop = asyncio.new_event_loop()       # 1. 创建事件循环
    try:
        asyncio.set_event_loop(loop)
        return loop.run_until_complete(main_coroutine)  # 2. 运行
    finally:
        loop.close()                      # 3. 清理
```

## 协程基础：`async` / `await`

### 定义和运行协程

`async def` 定义协程函数，`await` 等待异步操作完成：

```python
import asyncio

# async def 定义一个协程函数
async def greet(name):
    await asyncio.sleep(1)   # 模拟 IO 等待（比如网络请求）
    return f"Hello, {name}!"

# 调用协程函数返回一个协程对象（此时不会执行！）
coro = greet("Alice")
print(type(coro))  # <class 'coroutine'>

# 运行协程的方式：
# 1. asyncio.run() — 程序的顶层入口
result = asyncio.run(coro)
print(result)  # Hello, Alice!

# 2. await — 在另一个协程内部等待
async def main():
    result = await greet("Bob")
    print(result)
```

> **重要**：调用 `greet("Alice")` 不会执行函数体！它只创建了一个协程对象。必须用 `await` 或 `asyncio.run()` 才会真正执行。

### 并发执行多个协程

这是异步的核心价值——同时等待多个 IO 操作：

```python
import asyncio
import time

async def fetch_data(delay, data_id):
    await asyncio.sleep(delay)    # 模拟网络请求
    return f"数据 {data_id}"

async def main():
    start = time.perf_counter()

    # 顺序执行 — 总耗时 = 所有任务耗时之和
    # result1 = await fetch_data(1, 1)
    # result2 = await fetch_data(1, 2)
    # result3 = await fetch_data(1, 3)
    # 用时: ~3 秒（一个等完才等下一个）

    # 并发执行 — 总耗时 ≈ 最慢的任务
    results = await asyncio.gather(
        fetch_data(1, 1),
        fetch_data(1, 2),
        fetch_data(1, 3),
    )

    elapsed = time.perf_counter() - start
    print(f"结果: {results}")     # ['数据 1', '数据 2', '数据 3']
    print(f"用时: {elapsed:.2f}秒")  # ~1.01秒（三个同时等！）

asyncio.run(main())
```

> 3个各需1秒的任务：顺序执行要3秒，并发执行只要1秒——因为三个请求同时在等。

### `await` 到底做了什么？

```python
async def demo():
    print("A")
    await asyncio.sleep(1)        # ① 挂起当前协程
    print("B")                    # ② 1秒后恢复

# 执行流程：
# 1. 执行 print("A")
# 2. 遇到 await asyncio.sleep(1)：
#    a. 告诉事件循环："我要等1秒，这期间你可以做其他事"
#    b. 事件循环去调度其他就绪的任务
# 3. 1秒后，事件循环恢复此协程
# 4. 执行 print("B")
```

> `await` = 「我先歇会，你忙别的，完事了叫我」。这就是异步高效的秘密。

## 可等待对象（Awaitables）

`await` 后面可以接三种东西：

### 1. 协程对象（Coroutine）

```python
async def compute():
    return 42

async def main():
    coro = compute()         # 创建协程对象
    result = await coro      # await 它
    print(result)            # 42
```

### 2. Task 对象

```python
async def main():
    task = asyncio.create_task(
        asyncio.sleep(1)     # 创建任务，立刻开始调度
    )
    await task   # 等待 task 完成
```

### 3. Future 对象

```python
async def main():
    loop = asyncio.get_running_loop()
    future = loop.create_future()

    # 在回调中设置结果
    loop.call_later(1, future.set_result, "完成")

    result = await future
    print(result)  # "完成"（1秒后）
```

### 三者的关系

```
Awaitable（可等待对象）
├── Coroutine  — async def 的返回值，最常用
├── Task       — 把协程包装成可调度的任务（是 Future 的子类）
└── Future     — 低级对象，表示「未来某个时刻会有结果」
```

> 日常开发中：你主要用 **Coroutine**（自己写的 async 函数）和 **Task**（`create_task` 创建的）。Future 很少直接用。

## Task：并发执行协程

### 为什么需要 Task？

直接 `await` 一个协程是**顺序执行**的——等完一个才等下一个。用 `create_task` 把协程包装成 Task，事件循环就能**同时调度**多个任务。

### 创建 Task

```python
import asyncio

async def worker(name, delay):
    await asyncio.sleep(delay)
    print(f"{name} 完成")
    return name

async def main():
    # create_task 立刻把协程提交给事件循环
    task1 = asyncio.create_task(worker("A", 2))
    task2 = asyncio.create_task(worker("B", 1))

    # 在 Task 运行时可以做其他事
    print("等待任务完成...")

    # 等待所有 Task
    results = await asyncio.gather(task1, task2)
    print(f"结果: {results}")

# 输出：
# 等待任务完成...
# B 完成           ← B 只需1秒，先完成
# A 完成           ← A 需2秒，后完成
# 结果: ['A', 'B']
```

### TaskGroup（Python 3.11+ 推荐）

`TaskGroup` 是更安全的并发方式——一个任务出错，其他任务自动取消：

```python
async def worker(name, delay):
    await asyncio.sleep(delay)
    if name == "bad":
        raise ValueError(f"{name} 出错了")
    return name

async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            t1 = tg.create_task(worker("A", 1))
            t2 = tg.create_task(worker("bad", 0.5))  # 这个会出错
            t3 = tg.create_task(worker("C", 2))
    except Exception as e:
        print(f"捕获: {e}")

    # t3 也会被取消（TaskGroup 保证结构化并发）
    print(f"t1: {t1.cancelled()}, t3: {t3.cancelled()}")

asyncio.run(main())
```

### Task 控制：取消任务

```python
async def long_task():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        print("任务被取消，执行清理...")
        raise       # 必须重新抛出！

async def main():
    task = asyncio.create_task(long_task())
    await asyncio.sleep(0.1)

    task.cancel()     # 请求取消
    try:
        await task
    except asyncio.CancelledError:
        print("主协程：任务已取消")

asyncio.run(main())
# 输出:
# 任务被取消，执行清理...
# 主协程：任务已取消
```

### `gather()` vs `wait()`

```python
async def main():
    async def delayed(delay, result):
        await asyncio.sleep(delay)
        return result

    task1 = asyncio.create_task(delayed(1, "A"))
    task2 = asyncio.create_task(delayed(2, "B"))

    # gather：简单好用，返回结果列表
    results = await asyncio.gather(task1, task2)
    # → ['A', 'B']

    # wait：更细粒度的控制，能设置超时和完成条件
    done, pending = await asyncio.wait(
        [task1, task2],
        timeout=1.5,                       # 超时1.5秒
        return_when=asyncio.FIRST_COMPLETED # 第一个完成就返回
    )
```

| 方法 | 特点 | 适用场景 |
|------|------|---------|
| `gather()` | 简单，返回结果列表 | 大多数并发场景 |
| `wait()` | 可设超时、完成条件 | 需要精细控制 |

## 异步上下文管理器

### 什么是异步上下文管理器？

就是 `with` 的异步版本——`async with`。在进入和退出时做异步操作（如异步连接/断开数据库）。

### 基本语法

```python
class AsyncResource:
    async def __aenter__(self):
        print("异步获取资源...")
        await asyncio.sleep(0.1)    # 比如异步建立连接
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("异步释放资源...")
        await asyncio.sleep(0.1)    # 比如异步关闭连接
        return False

    async def use(self):
        print("使用资源")

async def main():
    async with AsyncResource() as res:
        await res.use()

# 等价于：
# res = await AsyncResource().__aenter__()
# try:
#     await res.use()
# finally:
#     await res.__aexit__(...)
```

### 实战：异步数据库连接

```python
import asyncio

class AsyncDatabase:
    def __init__(self, dsn):
        self.dsn = dsn
        self._conn = None

    async def __aenter__(self):
        print(f"连接数据库: {self.dsn}")
        await asyncio.sleep(0.1)  # 模拟连接
        self._conn = f"conn-{self.dsn}"
        return self._conn

    async def __aexit__(self, *args):
        print("关闭数据库连接")
        await asyncio.sleep(0.05)

async def main():
    async with AsyncDatabase("postgresql://...") as conn:
        print(f"使用 {conn} 执行查询...")
```

### `asynccontextmanager` 简化写法

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def managed_connection(dsn):
    # __aenter__ 部分（进入时执行）
    print(f"连接: {dsn}")
    conn = f"conn-{dsn}"
    try:
        yield conn                    # 把资源交给 async with 块
    finally:
        # __aexit__ 部分（退出时执行）
        print("关闭连接")

async def main():
    async with managed_connection("mysql://...") as conn:
        print(f"使用 {conn}")
```

## 异步迭代器和异步生成器

### 异步迭代器

普通迭代器 + `async`：每次取下一个值时可以等待异步操作：

```python
class AsyncRange:
    def __init__(self, start, end):
        self.start = start
        self.end = end

    def __aiter__(self):
        self.current = self.start
        return self

    async def __anext__(self):
        if self.current >= self.end:
            raise StopAsyncIteration
        await asyncio.sleep(0.1)     # 异步操作（比如从网络取数据）
        value = self.current
        self.current += 1
        return value

async def main():
    async for num in AsyncRange(0, 5):
        print(num)
```

### 异步生成器（async generator）

更简洁的写法，用 `async def` + `yield`：

```python
async def async_range(start, end):
    for i in range(start, end):
        await asyncio.sleep(0.1)    # 异步操作
        yield i

async def main():
    async for num in async_range(0, 5):
        print(num)

    # 异步列表推导（Python 3.6+）
    results = [num async for num in async_range(0, 3)]
    print(results)  # [0, 1, 2]
```

### 实战：分页 API 调用

```python
async def fetch_page(page_num):
    """模拟获取分页数据"""
    await asyncio.sleep(0.5)   # 模拟网络请求
    items = [f"page{page_num}-item{i}" for i in range(3)]
    has_more = page_num < 3
    return items, has_more

async def paginated_fetch():
    """异步生成器：逐页返回数据"""
    page = 1
    while True:
        items, has_more = await fetch_page(page)
        for item in items:
            yield item
        if not has_more:
            break
        page += 1

async def main():
    async for item in paginated_fetch():
        print(f"获取到: {item}")

# 输出:
# 获取到: page1-item0
# ...共9条数据
```

## 同步原语

异步编程中也需要协调多个协程，asyncio 提供了和 threading 对应的工具：

### Lock（锁）— 防止同时访问

```python
async def access_resource(lock, name):
    async with lock:              # 获取锁
        print(f"{name} 获取了锁")
        await asyncio.sleep(0.5)
        print(f"{name} 释放了锁")

async def main():
    lock = asyncio.Lock()
    await asyncio.gather(
        access_resource(lock, "A"),
        access_resource(lock, "B"),
        access_resource(lock, "C"),
    )
# A、B、C 依次获取锁，不会同时进入
```

### Semaphore（信号量）— 限制并发数

```python
async def bounded_request(sem, url):
    async with sem:               # 限制同时只有 N 个请求
        print(f"请求: {url}")
        await asyncio.sleep(0.5)
        return f"{url} 结果"

async def main():
    sem = asyncio.Semaphore(3)   # 最多同时3个

    urls = [f"https://api.example.com/{i}" for i in range(10)]
    tasks = [bounded_request(sem, url) for url in urls]
    results = await asyncio.gather(*tasks)
    print(f"完成 {len(results)} 个请求")
```

> Semaphore 是实际开发中最常用的——防止一次性发太多请求把服务器搞挂。

### Event（事件）— 等待通知

```python
async def waiter(event, name):
    print(f"{name} 等待事件...")
    await event.wait()            # 阻塞直到 event.set() 被调用
    print(f"{name} 收到事件！")

async def setter(event):
    await asyncio.sleep(1)
    print("触发事件！")
    event.set()                   # 通知所有等待者

async def main():
    event = asyncio.Event()
    await asyncio.gather(
        waiter(event, "A"),
        waiter(event, "B"),
        setter(event),
    )
```

### Queue（队列）— 生产者-消费者

```python
async def producer(queue, num_consumers):
    for i in range(5):
        await asyncio.sleep(0.5)
        await queue.put(f"消息 {i}")
        print(f"生产: 消息 {i}")
    for _ in range(num_consumers):
        await queue.put(None)     # 发送结束信号

async def consumer(queue, name):
    while True:
        item = await queue.get()
        if item is None:
            queue.task_done()
            break
        print(f"{name} 消费: {item}")
        await asyncio.sleep(1)
        queue.task_done()

async def main():
    queue = asyncio.Queue(maxsize=2)
    num_consumers = 2
    await asyncio.gather(
        producer(queue, num_consumers),
        consumer(queue, "C1"),
        consumer(queue, "C2"),
    )
```

### 同步原语对照表

| 线程 (threading) | 协程 (asyncio) | 用途 | 大白话 |
|-------------------|----------------|------|--------|
| `threading.Lock` | `asyncio.Lock` | 互斥锁 | 厕所门锁 |
| `threading.Semaphore` | `asyncio.Semaphore` | 限流 | 停车场车位 |
| `threading.Event` | `asyncio.Event` | 通知 | 下课铃 |
| `queue.Queue` | `asyncio.Queue` | 生产消费 | 传送带 |
| `threading.Condition` | `asyncio.Condition` | 条件变量 | 等条件满足 |

## 异步网络 IO

### TCP Echo 服务器

```python
import asyncio

async def handle_client(reader, writer):
    """处理单个客户端连接"""
    addr = writer.get_extra_info('peername')
    print(f"新连接: {addr}")

    while True:
        data = await reader.read(1024)    # 异步读取
        if not data:
            break
        message = data.decode()
        print(f"收到: {message}")
        writer.write(f"Echo: {message}".encode())
        await writer.drain()    # 确保数据已发送

    print(f"连接关闭: {addr}")
    writer.close()
    await writer.wait_closed()

async def main():
    server = await asyncio.start_server(
        handle_client, '127.0.0.1', 8888
    )
    addr = server.sockets[0].getsockname()
    print(f"服务器启动: {addr}")

    async with server:
        await server.serve_forever()
```

### HTTP 客户端（aiohttp）

```python
import aiohttp
import asyncio

async def fetch(session, url):
    async with session.get(url) as response:
        return await response.text()

async def main():
    async with aiohttp.ClientSession() as session:
        urls = [
            "https://httpbin.org/delay/1",
            "https://httpbin.org/delay/1",
            "https://httpbin.org/delay/1",
        ]
        tasks = [fetch(session, url) for url in urls]
        results = await asyncio.gather(*tasks)  # 3个请求并发
        print(f"完成 {len(results)} 个请求")

# asyncio.run(main())
```

> 用 `requests` 库是同步的，会阻塞事件循环。异步代码中必须用 `aiohttp` 或 `httpx` 等异步库。

## 与多线程/多进程的结合

### 在线程池中运行阻塞代码

有时候你不得不用同步的阻塞库（如 `requests`），怎么办？把它丢到线程池里：

```python
import asyncio
import time
from concurrent.futures import ThreadPoolExecutor

def blocking_io(n):
    """阻塞式 IO 操作（无法 await）"""
    time.sleep(1)        # 这会阻塞！
    return f"结果 {n}"

async def main():
    loop = asyncio.get_running_loop()

    # 方式 1：直接用（None = 默认线程池）
    result = await loop.run_in_executor(None, blocking_io, 42)

    # 方式 2：用自定义线程池
    with ThreadPoolExecutor() as pool:
        tasks = [
            loop.run_in_executor(pool, blocking_io, i)
            for i in range(5)
        ]
        results = await asyncio.gather(*tasks)
        print(results)  # 5个结果，总用时约1秒（并发）
```

### CPU 密集型任务放到进程池

异步不适合 CPU 密集型任务，但你可以用进程池来处理：

```python
import asyncio
from concurrent.futures import ProcessPoolExecutor

def cpu_intensive(n):
    """CPU 密集型计算"""
    return sum(i * i for i in range(n))

async def main():
    loop = asyncio.get_running_loop()

    with ProcessPoolExecutor() as pool:
        tasks = [
            loop.run_in_executor(pool, cpu_intensive, 10_000_000)
            for _ in range(4)
        ]
        results = await asyncio.gather(*tasks)
```

### `asyncio.to_thread()`（Python 3.9+ 更简洁）

```python
import asyncio
import time

def blocking_function():
    time.sleep(1)
    return "完成"

async def main():
    # 便捷方法：在单独的线程中运行阻塞函数
    result = await asyncio.to_thread(blocking_function)
    print(result)
```

## 生成器 vs 协程

### 它们很像，但有区别

生成器和协程都能「暂停和恢复」，但用途不同：

```python
# 生成器：通过 yield 暂停，用于生产数据
def generator():
    yield 1
    yield 2
    yield 3

# 协程：通过 await 暂停，用于等待 IO
async def coroutine():
    await asyncio.sleep(0)
    return 42
```

### 关键区别

| 特性 | 生成器 | 协程 |
|------|--------|------|
| 定义 | `def` + `yield` | `async def` + `await` |
| 调用结果 | `generator` 对象 | `coroutine` 对象 |
| 推进方式 | `next()` / `send()` | `await` / `asyncio.run()` |
| 暂停方式 | `yield` | `await` |
| 数据流向 | 可以吐值出来 + 接收值进去 | 只能等待结果 |
| 嵌套 | `yield from` | `await` |
| 用途 | 惰性生产数据 | 异步 IO 并发 |

> 简单记：**生成器是「生产者」**（一个一个吐数据），**协程是「等待者」**（等 IO 完成后继续）。

### 为什么需要专门的 async/await？

1. **语义清晰**：生成器用于迭代数据；协程用于异步 IO，不会混淆
2. **类型安全**：不会把迭代器接口和异步接口搞混
3. **更好的报错**：`await` 在非协程中会报错，`yield` 不会
4. **性能优化**：原生协程比生成器协程更快

## 常见陷阱与最佳实践

### 1. 不要在协程中使用阻塞调用（最常见的坑！）

```python
# ❌ 危险：time.sleep() 会阻塞整个事件循环！所有协程都被卡住
async def bad():
    time.sleep(5)

# ✅ 正确：用 asyncio.sleep()
async def good():
    await asyncio.sleep(5)

# 规则：异步代码中不能用 time.sleep、requests.get、普通 open() 大文件
# 必须用 await 版本，或者放到 run_in_executor 里
```

### 2. 不要忘记 await（第二常见的坑！）

```python
# ❌ 忘记 await — 协程不会执行！
async def main():
    asyncio.sleep(1)          # 创建了协程对象但没有执行
    # RuntimeWarning: coroutine 'sleep' was never awaited

# ✅ 正确
async def main():
    await asyncio.sleep(1)
```

> 忘记 `await` 就像点了外卖但没付钱——订单创建了，但永远不会送到。

### 3. 正确处理 CancelledError

```python
# ❌ 吞掉取消信号，任务无法被取消
async def bad():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        print("忽略取消")     # 吞掉了，无法取消！

# ✅ 必须重新抛出
async def good():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        await cleanup()       # 执行清理
        raise                 # 传播取消信号
```

### 4. Task 引用防止被 GC 回收

```python
# ❌ 危险：创建的 Task 没有引用，可能被垃圾回收
async def bad():
    asyncio.create_task(important_work())  # 没人引用，可能被回收

# ✅ 保留引用
async def good():
    task = asyncio.create_task(important_work())
    # ... 或者使用 TaskGroup
```

### 5. 使用超时机制

```python
async def main():
    try:
        async with asyncio.timeout(5):     # Python 3.11+
            result = await slow_operation()
    except TimeoutError:
        print("操作超时！")

    # 旧版本写法
    # result = await asyncio.wait_for(slow_operation(), timeout=5)
```

### 6. 并发网络请求的限流

```python
async def fetch_with_limit(urls, concurrency=10):
    """限制并发数的批量请求——防止一次性发太多请求"""
    sem = asyncio.Semaphore(concurrency)

    async def bounded_fetch(url):
        async with sem:
            return await fetch_url(url)

    return await asyncio.gather(*[bounded_fetch(u) for u in urls])
```

### 7. 优雅关闭

```python
async def main():
    import signal
    loop = asyncio.get_running_loop()
    server = await asyncio.start_server(handler, '0.0.0.0', 8888)

    async def shutdown():
        server.close()
        await server.wait_closed()

    for sig in (signal.SIGTERM, signal.SIGINT):
        loop.add_signal_handler(
            sig, lambda: asyncio.create_task(shutdown())
        )

    async with server:
        await server.serve_forever()
```

### 8. 优先用 TaskGroup（Python 3.11+）

```python
# ✅ 推荐：TaskGroup 自动管理，一个出错全取消
async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(work("A"))
            tg.create_task(work("B"))
            tg.create_task(work("C"))
    except Exception:
        pass

# ❌ 不推荐：手动管理，容易遗漏清理
async def old_style():
    tasks = []
    try:
        for name in ["A", "B", "C"]:
            tasks.append(asyncio.create_task(work(name)))
        await asyncio.gather(*tasks)
    except Exception:
        for t in tasks:
            t.cancel()
        raise
```

## 总结

| 概念 | 一句话解释 | 类比 |
|------|-----------|------|
| `async def` | 定义协程函数 | 声明「我是异步的」 |
| `await` | 挂起等待，让出 CPU | 「我先等会，你忙别的」 |
| `asyncio.run()` | 启动事件循环 | 打开总开关 |
| `create_task()` | 把协程包装成可调度任务 | 把任务排进日程表 |
| `asyncio.gather()` | 同时等多个任务 | 同时点多份外卖 |
| `TaskGroup` | 更安全的并发管理 | 一个出错全停 |
| `asyncio.Queue` | 协程间传数据 | 传送带 |
| `run_in_executor()` | 让阻塞代码不卡住 | 把脏活丢给别人 |
| `asyncio.timeout()` | 设置超时 | 等不及就走 |

异步编程的核心心智模型：**事件循环在单线程里调度协程，协程遇到 IO 等待就主动让出控制权（通过 `await`），从而在等待时去做其他事，实现高并发。** 适合 IO 密集型任务，不适合 CPU 密集型任务（后者用多进程）。
