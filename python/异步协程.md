## 概述

### 为什么需要异步编程？

| 模型 | 原理 | 适用场景 | 切换成本 |
|------|------|---------|---------|
| 多线程 | OS 调度，共享内存 | CPU 密集型 | 高（上下文切换） |
| 多进程 | 独立内存空间，IPC 通信 | CPU 密集型 | 很高 |
| **异步** | 单线程协作式，用户态调度 | **IO 密集型** | 极低 |

异步编程解决的核心问题是**IO 密集型任务**中大量时间浪费在等待上（网络请求、文件读写、数据库查询），通过协作式调度，在等待 IO 时切换到其他任务，充分利用 CPU。

### Python 异步演变史

```
Python 2:  回调地狱（callback hell）
    ↓
Python 3.3: yield from（生成器协程）
    ↓
Python 3.4: asyncio 库 + @asyncio.coroutine
    ↓
Python 3.5: async / await 语法（原生协程）
    ↓
Python 3.7+: asyncio 高级 API 完善
    ↓
Python 3.11+: TaskGroup、Runner、性能大幅优化
```

## 事件循环（Event Loop）

事件循环是异步编程的核心引擎，负责调度和执行协程。

### 工作原理

```
┌─────────────────────────────────────────┐
│              Event Loop                  │
│                                          │
│   ┌──────┐    ┌──────┐    ┌──────┐      │
│   │ Task1│    │ Task2│    │ Task3│       │
│   │ 运行 │    │ 等待 │    │ 就绪 │       │
│   └──┬───┘    └──┬───┘    └──┬───┘      │
│      │  await    │           │           │
│      ▼           ▼           ▼           │
│   ┌──────────────────────────────┐      │
│   │       IO 等待队列             │      │
│   │   (epoll/kqueue/IOCP)        │      │
│   └──────────────────────────────┘      │
└─────────────────────────────────────────┘
```

### 运行事件循环

```python
import asyncio

async def main():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

# Python 3.7+ 推荐方式
asyncio.run(main())
```

> `asyncio.run()` 创建事件循环、运行协程、并在完成后清理。每个线程同时只能有一个事件循环运行。

### 事件循环的生命周期

```python
# asyncio.run() 的简化等价实现
def my_run(main_coroutine):
    loop = asyncio.new_event_loop()
    try:
        asyncio.set_event_loop(loop)
        return loop.run_until_complete(main_coroutine)
    finally:
        loop.close()
```

## 协程基础：`async` / `await`

### 定义和运行协程

```python
import asyncio

# async def 定义一个协程函数
async def greet(name):
    await asyncio.sleep(1)   # 模拟 IO 等待
    return f"Hello, {name}!"

# 调用协程函数返回一个协程对象（此时不会执行！）
coro = greet("Alice")
print(type(coro))  # <class 'coroutine'>

# 运行协程的方式：
# 1. asyncio.run() — 顶层入口
result = asyncio.run(coro)
print(result)  # Hello, Alice!

# 2. await — 在另一个协程中等待
async def main():
    result = await greet("Bob")
    print(result)
```

### 并发执行多个协程

```python
import asyncio
import time

async def fetch_data(delay, data_id):
    await asyncio.sleep(delay)
    return f"数据 {data_id}"

async def main():
    start = time.perf_counter()

    # 顺序执行 — 总耗时 = 所有任务耗时之和
    # result1 = await fetch_data(1, 1)
    # result2 = await fetch_data(1, 2)
    # result3 = await fetch_data(1, 3)
    # 用时: ~3 秒

    # 并发执行 — 总耗时 ≈ 最慢的任务
    results = await asyncio.gather(
        fetch_data(1, 1),
        fetch_data(1, 2),
        fetch_data(1, 3),
    )

    elapsed = time.perf_counter() - start
    print(f"结果: {results}")     # ['数据 1', '数据 2', '数据 3']
    print(f"用时: {elapsed:.2f}秒")  # ~1.01秒

asyncio.run(main())
```

### `await` 到底做了什么？

```python
async def demo():
    print("A")
    await asyncio.sleep(1)        # ① 挂起当前协程
    print("B")                    # ② 1秒后恢复

# 执行流程：
# 1. 执行 print("A")
# 2. 遇到 await asyncio.sleep(1)：
#    a. 告诉事件循环："我要睡1秒，这期间你可以做其他事"
#    b. 事件循环去调度其他就绪的任务
# 3. 1秒后，事件循环恢复此协程
# 4. 执行 print("B")
```

## 可等待对象（Awaitables）

`await` 后面可以接三种对象：

### 1. 协程对象（Coroutine）

```python
async def compute():
    return 42

async def main():
    coro = compute()         # 创建协程对象
    result = await coro      # await 它
    print(result)            # 42
```

### 2. Task 对象

```python
async def main():
    task = asyncio.create_task(
        asyncio.sleep(1)
    )
    await task   # 等待 task 完成
```

### 3. Future 对象

```python
async def main():
    loop = asyncio.get_running_loop()
    future = loop.create_future()

    # 在回调中设置结果
    loop.call_later(1, future.set_result, "完成")

    result = await future
    print(result)  # "完成"（1秒后）
```

### 可等待对象关系

```
Awaitable
├── Coroutine        — async def 的返回值
├── Task             — 对 Coroutine 的调度包装（是 Future 的子类）
└── Future           — 低级可等待对象
```

## Task：并发执行协程

Task 是事件循环调度协程的单元，允许多个协程并发执行。

### 创建 Task

```python
import asyncio

async def worker(name, delay):
    await asyncio.sleep(delay)
    print(f"{name} 完成")
    return name

async def main():
    # 方式 1：asyncio.create_task()
    task1 = asyncio.create_task(worker("A", 2))
    task2 = asyncio.create_task(worker("B", 1))

    # 在 Task 运行时可以做其他事
    print("等待任务完成...")

    # 等待所有 Task
    results = await asyncio.gather(task1, task2)
    print(f"结果: {results}")

# 输出：
# 等待任务完成...
# B 完成
# A 完成
# 结果: ['A', 'B']
```

### TaskGroup（Python 3.11+）

```python
async def worker(name, delay):
    await asyncio.sleep(delay)
    if name == "bad":
        raise ValueError(f"{name} 出错了")
    return name

async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            t1 = tg.create_task(worker("A", 1))
            t2 = tg.create_task(worker("bad", 0.5))
            t3 = tg.create_task(worker("C", 2))
    except Exception as e:
        print(f"捕获: {e}")

    # t3 也会被取消（TaskGroup 保证结构化并发）
    print(f"t1: {t1.cancelled()}, t3: {t3.cancelled()}")

asyncio.run(main())
```

### Task 控制

```python
async def long_task():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        print("任务被取消，执行清理...")
        raise       # 必须重新抛出！

async def main():
    task = asyncio.create_task(long_task())
    await asyncio.sleep(0.1)

    task.cancel()     # 请求取消
    try:
        await task
    except asyncio.CancelledError:
        print("主协程：任务已取消")

asyncio.run(main())
# 输出:
# 任务被取消，执行清理...
# 主协程：任务已取消
```

### `asyncio.gather()` vs `asyncio.wait()`

```python
async def main():
    async def delayed(delay, result):
        await asyncio.sleep(delay)
        return result

    task1 = asyncio.create_task(delayed(1, "A"))
    task2 = asyncio.create_task(delayed(2, "B"))

    # gather：返回结果列表，可通过 return_exceptions 控制异常
    results = await asyncio.gather(task1, task2)
    # → ['A', 'B']

    # wait：更细粒度的控制
    done, pending = await asyncio.wait(
        [task1, task2],
        timeout=1.5,                       # 超时
        return_when=asyncio.FIRST_COMPLETED # 首个完成就返回
    )
```

## 异步上下文管理器

### 基本语法

```python
class AsyncResource:
    async def __aenter__(self):
        print("异步获取资源...")
        await asyncio.sleep(0.1)
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("异步释放资源...")
        await asyncio.sleep(0.1)
        return False     # False = 不抑制异常

    async def use(self):
        print("使用资源")

async def main():
    async with AsyncResource() as res:
        await res.use()

# 等价于：
# res = await AsyncResource().__aenter__()
# try:
#     await res.use()
# finally:
#     await res.__aexit__(...)
```

### 实战：异步数据库连接

```python
import asyncio

class AsyncDatabase:
    def __init__(self, dsn):
        self.dsn = dsn
        self._conn = None

    async def __aenter__(self):
        print(f"连接数据库: {self.dsn}")
        await asyncio.sleep(0.1)  # 模拟连接
        self._conn = f"conn-{self.dsn}"
        return self._conn

    async def __aexit__(self, *args):
        print("关闭数据库连接")
        await asyncio.sleep(0.05)

async def main():
    async with AsyncDatabase("postgresql://...") as conn:
        print(f"使用 {conn} 执行查询...")
```

### `asynccontextmanager`

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def managed_connection(dsn):
    # __aenter__ 部分
    print(f"连接: {dsn}")
    conn = f"conn-{dsn}"
    try:
        yield conn                    # 资源提供给 async with 块
    finally:
        # __aexit__ 部分
        print("关闭连接")

async def main():
    async with managed_connection("mysql://...") as conn:
        print(f"使用 {conn}")
```

## 异步迭代器和异步生成器

### 异步迭代器

```python
class AsyncRange:
    def __init__(self, start, end):
        self.start = start
        self.end = end

    def __aiter__(self):
        self.current = self.start
        return self

    async def __anext__(self):
        if self.current >= self.end:
            raise StopAsyncIteration
        await asyncio.sleep(0.1)     # 异步操作
        value = self.current
        self.current += 1
        return value

async def main():
    async for num in AsyncRange(0, 5):
        print(num)
```

### 异步生成器（async generator）

```python
async def async_range(start, end):
    for i in range(start, end):
        await asyncio.sleep(0.1)
        yield i

async def main():
    async for num in async_range(0, 5):
        print(num)

    # 异步列表推导（Python 3.6+）
    results = [num async for num in async_range(0, 3)]
    print(results)  # [0, 1, 2]
```

### 实战：分页 API 调用

```python
async def fetch_page(page_num):
    """模拟获取分页数据"""
    await asyncio.sleep(0.5)   # 模拟网络请求
    items = [f"page{page_num}-item{i}" for i in range(3)]
    has_more = page_num < 3
    return items, has_more

async def paginated_fetch():
    """异步生成器：逐页返回数据"""
    page = 1
    while True:
        items, has_more = await fetch_page(page)
        for item in items:
            yield item
        if not has_more:
            break
        page += 1

async def main():
    async for item in paginated_fetch():
        print(f"获取到: {item}")

# 输出:
# 获取到: page1-item0
# ...共9条数据
```

## 同步原语

asyncio 提供了与 threading 模块对应的同步原语：

### Lock

```python
async def access_resource(lock, name):
    async with lock:
        print(f"{name} 获取了锁")
        await asyncio.sleep(0.5)
        print(f"{name} 释放了锁")

async def main():
    lock = asyncio.Lock()
    await asyncio.gather(
        access_resource(lock, "A"),
        access_resource(lock, "B"),
        access_resource(lock, "C"),
    )
```

### Semaphore（限流）

```python
async def bounded_request(sem, url):
    async with sem:
        print(f"请求: {url}")
        await asyncio.sleep(0.5)  # 模拟请求
        return f"{url} 结果"

async def main():
    # 最多同时 3 个请求
    sem = asyncio.Semaphore(3)

    urls = [f"https://api.example.com/{i}" for i in range(10)]
    tasks = [bounded_request(sem, url) for url in urls]
    results = await asyncio.gather(*tasks)
    print(f"完成 {len(results)} 个请求")
```

### Event

```python
async def waiter(event, name):
    print(f"{name} 等待事件...")
    await event.wait()
    print(f"{name} 收到事件！")

async def setter(event):
    await asyncio.sleep(1)
    print("触发事件！")
    event.set()

async def main():
    event = asyncio.Event()
    await asyncio.gather(
        waiter(event, "A"),
        waiter(event, "B"),
        setter(event),
    )
```

### Queue

```python
async def producer(queue, num_consumers):
    for i in range(5):
        await asyncio.sleep(0.5)
        await queue.put(f"消息 {i}")
        print(f"生产: 消息 {i}")
    # 每个消费者都需要一个结束信号
    for _ in range(num_consumers):
        await queue.put(None)

async def consumer(queue, name):
    while True:
        item = await queue.get()
        if item is None:
            queue.task_done()
            break
        print(f"{name} 消费: {item}")
        await asyncio.sleep(1)  # 模拟处理
        queue.task_done()

async def main():
    queue = asyncio.Queue(maxsize=2)
    num_consumers = 2
    await asyncio.gather(
        producer(queue, num_consumers),
        consumer(queue, "C1"),
        consumer(queue, "C2"),
    )
```

### 同步原语对照表

| 线程 (threading) | 协程 (asyncio) | 用途 |
|-------------------|----------------|------|
| `threading.Lock` | `asyncio.Lock` | 互斥锁 |
| `threading.Semaphore` | `asyncio.Semaphore` | 信号量限流 |
| `threading.Event` | `asyncio.Event` | 事件通知 |
| `queue.Queue` | `asyncio.Queue` | 生产者-消费者 |
| `threading.Condition` | `asyncio.Condition` | 条件变量 |

## 异步网络 IO

### TCP Echo 服务器

```python
import asyncio

async def handle_client(reader, writer):
    """处理单个客户端连接"""
    addr = writer.get_extra_info('peername')
    print(f"新连接: {addr}")

    while True:
        data = await reader.read(1024)
        if not data:
            break
        message = data.decode()
        print(f"收到: {message}")
        writer.write(f"Echo: {message}".encode())
        await writer.drain()    # 确保数据已发送

    print(f"连接关闭: {addr}")
    writer.close()
    await writer.wait_closed()

async def main():
    server = await asyncio.start_server(
        handle_client, '127.0.0.1', 8888
    )
    addr = server.sockets[0].getsockname()
    print(f"服务器启动: {addr}")

    async with server:
        await server.serve_forever()
```

## 与多线程/多进程的结合

### 在线程池中运行阻塞代码

```python
import asyncio
import time
from concurrent.futures import ThreadPoolExecutor

def blocking_io(n):
    """阻塞式 IO 操作（无法 await）"""
    time.sleep(1)
    return f"结果 {n}"

async def main():
    loop = asyncio.get_running_loop()

    # 方式 1：直接使用
    result = await loop.run_in_executor(None, blocking_io, 42)

    # 方式 2：使用线程池
    with ThreadPoolExecutor() as pool:
        tasks = [
            loop.run_in_executor(pool, blocking_io, i)
            for i in range(5)
        ]
        results = await asyncio.gather(*tasks)
        print(results)  # 5个结果，总用时约1秒（并发）
```

### CPU 密集型任务放到进程池

```python
import asyncio
from concurrent.futures import ProcessPoolExecutor

def cpu_intensive(n):
    """CPU 密集型计算"""
    return sum(i * i for i in range(n))

async def main():
    loop = asyncio.get_running_loop()

    with ProcessPoolExecutor() as pool:
        tasks = [
            loop.run_in_executor(pool, cpu_intensive, 10_000_000)
            for _ in range(4)
        ]
        results = await asyncio.gather(*tasks)
```

### `asyncio.to_thread()`（Python 3.9+）

```python
import asyncio
import time

def blocking_function():
    time.sleep(1)
    return "完成"

async def main():
    # 便捷方法：在单独的线程中运行
    result = await asyncio.to_thread(blocking_function)
    print(result)
```

## 生成器 vs 协程

### 相似点

两者都支持暂停和恢复执行：

```python
# 生成器：通过 yield 暂停
def generator():
    yield 1
    yield 2
    yield 3

# 协程：通过 await 暂停
async def coroutine():
    await asyncio.sleep(0)
    return 42
```

### 关键区别

| 特性 | 生成器 | 协程 |
|------|--------|------|
| 定义 | `def` + `yield` | `async def` + `await` |
| 调用结果 | `generator` 对象 | `coroutine` 对象 |
| 推进方式 | `next()` / `send()` | `await` / `asyncio.run()` |
| 暂停方式 | `yield` | `await` |
| 数据流向 | 可以 `yield` 出 + `send` 入 | 只能 `await` 等待结果 |
| 嵌套 | `yield from` | `await` |
| 用途 | 惰性数据生产 | 异步 IO 并发 |

### 基于生成器的协程（历史遗留）

```python
# Python 3.4 时代的方式（已过时，不要在新代码中使用）
@asyncio.coroutine
def old_style():
    data = yield from fetch_data()
    return data

# 现代方式
async def new_style():
    data = await fetch_data()
    return data
```

### 为什么需要专门的 async/await？

1. **语义清晰**：生成器用于迭代数据；协程用于异步 IO
2. **类型安全**：不会混淆迭代器接口和异步接口
3. **更好的报错**：`await` 在非协程中会报错，`yield` 不会
4. **性能优化**：原生协程比生成器协程更快

## 常见陷阱与最佳实践

### 1. 不要在协程中使用阻塞调用

```python
# ❌ 危险：time.sleep() 会阻塞整个事件循环！
async def bad():
    time.sleep(5)

# ✅ 正确：使用 await asyncio.sleep()
async def good():
    await asyncio.sleep(5)

# 规则：永远不要在不使用 run_in_executor 的情况下
# 调用阻塞函数（time.sleep, requests.get, open() 大文件等）
```

### 2. 不要忘记 await

```python
# ❌ 忘记 await — 协程不会执行！
async def main():
    asyncio.sleep(1)          # 创建了协程对象但没有执行
    # RuntimeWarning: coroutine 'sleep' was never awaited

# ✅ 正确
async def main():
    await asyncio.sleep(1)
```

### 3. 正确处理 CancelledError

```python
# ❌ 吞掉 CancelledError
async def bad():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        print("忽略取消")     # 任务被吞掉，无法取消！

# ✅ 必须重新抛出
async def good():
    try:
        await asyncio.sleep(10)
    except asyncio.CancelledError:
        await cleanup()       # 执行清理
        raise                 # 传播取消信号
```

### 4. Task 引用防止被 GC 回收

```python
# ❌ 危险：创建的 Task 可能被垃圾回收
async def bad():
    asyncio.create_task(important_work())  # 无引用

# ✅ 保留引用
async def good():
    task = asyncio.create_task(important_work())
    # ... 或者使用 TaskGroup
```

### 5. 使用超时机制

```python
async def main():
    try:
        async with asyncio.timeout(5):     # Python 3.11+
            result = await slow_operation()
    except TimeoutError:
        print("操作超时！")

    # 旧版本写法
    # result = await asyncio.wait_for(slow_operation(), timeout=5)
```

### 6. 并发网络请求的限流

```python
async def fetch_with_limit(urls, concurrency=10):
    """限制并发数的批量请求"""
    sem = asyncio.Semaphore(concurrency)

    async def bounded_fetch(url):
        async with sem:
            return await fetch_url(url)

    return await asyncio.gather(*[bounded_fetch(u) for u in urls])
```

### 7. 优雅关闭

```python
async def main():
    import signal
    loop = asyncio.get_running_loop()
    server = await asyncio.start_server(handler, '0.0.0.0', 8888)

    # 关闭逻辑
    async def shutdown():
        server.close()
        await server.wait_closed()

    # 注册信号处理
    for sig in (signal.SIGTERM, signal.SIGINT):
        loop.add_signal_handler(
            sig, lambda: asyncio.create_task(shutdown())
        )

    async with server:
        await server.serve_forever()
```

### 8. TaskGroup 结构化并发（Python 3.11+ 推荐）

```python
# ✅ 推荐：TaskGroup 保证结构化并发
async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(work("A"))
            tg.create_task(work("B"))
            tg.create_task(work("C"))
    except Exception:
        # 任意任务出错，其他任务自动取消
        pass

# ❌ 不推荐：手动管理多个 create_task
async def old_style():
    tasks = []
    try:
        for name in ["A", "B", "C"]:
            tasks.append(asyncio.create_task(work(name)))
        await asyncio.gather(*tasks)
    except Exception:
        for t in tasks:
            t.cancel()      # 手动清理，容易遗漏
        raise
```

---

## 总结

| 概念 | 作用 |
|------|------|
| `async def` | 定义协程函数 |
| `await` | 挂起协程，等待可等待对象完成 |
| `asyncio.run()` | 运行协程的入口 |
| `create_task()` | 将协程包装为 Task 以并发执行 |
| `asyncio.gather()` | 并发运行多个 awaitable |
| `TaskGroup` | 结构化并发（3.11+） |
| `asyncio.Queue` | 协程间的生产者-消费者队列 |
| `run_in_executor()` | 在线程/进程池中运行阻塞代码 |
| `asyncio.timeout()` | 设置超时（3.11+） |

异步编程的核心心智模型是：**事件循环在用户态调度协程，协程在 IO 等待时主动让出控制权（通过 await），从而在单线程中实现高并发。** 适合 IO 密集型任务，但不适合 CPU 密集型任务（后者应使用多进程）。
